# LiveDesk Submission README

## Project

LiveDesk is a real-time browser-based video support platform. A call agent creates a support session, shares an invite link with a customer, and both join from a browser without installing any app.

## Source Code

```text
https://github.com/aditya-1408/LiveDesk
```

## Important Architecture Note

LiveDesk uses **self-hosted open-source mediasoup SFU** for audio/video routing.

It does **not** use LiveKit Cloud, Twilio, Agora, Daily, or any hosted video SDK. Media is routed through our own Node.js mediasoup server, so the call is not direct peer-to-peer.

## Local Setup

Requirements:

- Node.js 20+ or 22+
- Chrome or Edge browser
- For real two-device testing: laptop and phone connected to the same Wi-Fi

Clone the repository:

```bash
git clone https://github.com/aditya-1408/LiveDesk.git
cd LiveDesk
```

Install dependencies:

```bash
npm install --prefix server
npm install --prefix client
```

Create backend environment file:

```bash
cd server
copy .env.example .env
```

For phone testing, update `server/.env`:

```text
ANNOUNCED_IP=YOUR_LAPTOP_WIFI_IPV4
```

Also ensure `CLIENT_ORIGINS` includes:

```text
https://YOUR_LAPTOP_WIFI_IPV4:5173
```

On Windows, find laptop IP using:

```bash
ipconfig
```

Use the Wi-Fi IPv4 address, for example:

```text
10.62.43.31
```

## Start Commands

Terminal 1 - backend:

```bash
cd server
npm start
```

Terminal 2 - frontend:

```bash
cd client
npm run dev:https
```

Open on laptop:

```text
https://localhost:5173
```

Open on phone:

```text
https://YOUR_LAPTOP_WIFI_IPV4:5173
```

If the phone shows an HTTPS certificate warning, choose Advanced and proceed.

## Evaluation Flow

1. Open `https://localhost:5173/login` on laptop.
2. Select `Call Agent`.
3. Login using agent credentials.
4. Click `New session`.
5. Copy the customer invite link.
6. Open the invite link on phone.
7. Replace `localhost` in the invite URL with the laptop Wi-Fi IP.
8. Allow camera and microphone on both laptop and phone.
9. Join as customer on phone.
10. Join as agent on laptop.
11. Verify two-way audio/video.
12. Test mute and camera-off indicators.
13. Send chat messages both ways.
14. Upload a file in chat.
15. Start and stop recording from agent side.
16. Download the final recording.
17. Test customer leaving the call and reconnecting within the 5-minute window.
18. End the session from agent side.
19. Login as admin and verify live/history dashboard.
20. Open metrics endpoint and verify active sessions/participants.

## Implemented Features

- Session creation by call agent
- Shareable customer invite link/token
- Browser join with no app installation
- Real-time server-routed audio/video through mediasoup SFU
- Presence tracking
- Clean session end and socket/media cleanup
- Customer leave with 5-minute reconnect window
- Persisted session history with participants, events, and duration
- Realtime in-call chat
- Persisted chat history
- File sharing in chat
- Agent/admin/customer role separation
- Agent-controlled recording with final downloadable WebM
- Admin dashboard for live sessions and history
- Admin ability to end active sessions
- JSON and Prometheus observability metrics

## Known Limitations

- Full public deployment was not provided because this project uses self-hosted mediasoup, which requires a public server/VPS with WebRTC RTP ports open. A free static hosting platform such as Vercel or Netlify is not enough for this architecture.
- A full public deployment would require a VPS/public server with:
  - TCP 80/443 for HTTPS
  - UDP 40000-40100 for WebRTC media
  - TCP 40000-40100 for WebRTC fallback
  - `ANNOUNCED_IP` set to the public server IP
- VPS hosting may cost money, so for the hackathon submission this project is provided with local setup instructions and a working demo video.
- SQLite and a single mediasoup worker are used for hackathon evaluation, not horizontal production scaling.
- Recording is implemented through the agent browser using composed MediaRecorder output and then uploaded to the server. A production SFU-side recorder would pipe mediasoup RTP into ffmpeg.
- For best real-device testing, use laptop plus phone on the same Wi-Fi.
